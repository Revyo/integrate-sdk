# Configuring Additional Integrations

The Integrate SDK can be configured to work with any integration supported by the Integrate MCP server. While GitHub and Gmail have pre-built typed client interfaces, you can configure OAuth and enable tools for other server-supported integrations.

**Important**: Plugins configure OAuth credentials and enable server-provided tools. They do NOT create new tool implementations. All tools must exist on the Integrate MCP server.

## Using the Generic OAuth Plugin

To configure a plugin for any server-supported integration, use the `genericOAuthPlugin` helper:

```typescript
import { createMCPClient, genericOAuthPlugin } from "integrate-sdk";

// Configure Slack integration (assuming server supports Slack tools)
const slackPlugin = genericOAuthPlugin({
  id: "slack",
  provider: "slack",
  clientId: process.env.SLACK_CLIENT_ID!,
  clientSecret: process.env.SLACK_CLIENT_SECRET!,
  scopes: ["chat:write", "channels:read", "users:read"],
  tools: [
    "slack_send_message",    // Must exist on server
    "slack_list_channels",   // Must exist on server
    "slack_get_channel",     // Must exist on server
    "slack_invite_user",     // Must exist on server
  ],
  redirectUri: "https://your-app.com/callback",
});

const client = createMCPClient({
  plugins: [slackPlugin],
});

await client.connect();

// Call tools using _callToolByName
await client._callToolByName("slack_send_message", {
  channel: "#general",
  text: "Hello from Integrate SDK!",
});
```

## Generic OAuth Plugin Options

| Option         | Type     | Required | Description                                            |
| -------------- | -------- | -------- | ------------------------------------------------------ |
| `id`           | string   | Yes      | Plugin identifier (must match server integration ID)   |
| `provider`     | string   | Yes      | OAuth provider name (e.g., 'slack', 'notion')          |
| `clientId`     | string   | Yes      | OAuth client ID                                        |
| `clientSecret` | string   | Yes      | OAuth client secret                                    |
| `scopes`       | string[] | Yes      | OAuth scopes                                           |
| `tools`        | string[] | Yes      | Tool names to enable (must exist on server)            |
| `redirectUri`  | string   | No       | OAuth redirect URI                                     |

## Configuring Tools Without OAuth

For server-provided tools that don't require OAuth authentication, use `createSimplePlugin`:

```typescript
import { createSimplePlugin } from "integrate-sdk";

// Enable server-provided math tools (if they exist on the server)
const mathPlugin = createSimplePlugin({
  id: "math",
  tools: ["math_add", "math_subtract", "math_multiply", "math_divide"],
  onInit: async (client) => {
    console.log("Math plugin initialized");
  },
});

const client = createMCPClient({
  plugins: [mathPlugin],
});

await client.connect();

// Call tools using _callToolByName
await client._callToolByName("math_add", { a: 5, b: 3 });
```

## Implementing the MCPPlugin Interface

For more control over plugin configuration and lifecycle hooks, implement the `MCPPlugin` interface directly:

```typescript
import type { MCPPlugin } from "integrate-sdk";

interface NotionPluginConfig {
  clientId: string;
  clientSecret: string;
  scopes?: string[];
}

export function notionPlugin(config: NotionPluginConfig): MCPPlugin {
  return {
    id: "notion",
    tools: [
      "notion_search",
      "notion_create_page",
      "notion_update_page",
      "notion_query_database",
    ],
    oauth: {
      provider: "notion",
      clientId: config.clientId,
      clientSecret: config.clientSecret,
      scopes: config.scopes || ["read_content", "update_content"],
    },

    async onInit(client) {
      console.log("Notion plugin initialized");
    },

    async onBeforeConnect(client) {
      console.log("Notion plugin: preparing to connect");
    },

    async onAfterConnect(client) {
      console.log("Notion plugin: connected successfully");

      // Verify tools are available
      const tools = client.getEnabledTools();
      console.log("Notion tools available:", tools.length);
    },

    async onDisconnect(client) {
      console.log("Notion plugin: disconnected");
    },
  };
}
```

## Plugin Interface

```typescript
interface MCPPlugin {
  id: string; // Plugin identifier (must match server integration)
  tools: string[]; // Tool names to enable (must exist on server)
  oauth?: OAuthConfig; // OAuth configuration (optional)
  onInit?: PluginHook; // Called when plugin is created
  onBeforeConnect?: PluginHook; // Called before connecting
  onAfterConnect?: PluginHook; // Called after connecting
  onDisconnect?: PluginHook; // Called on disconnect
}

interface OAuthConfig {
  provider: string; // OAuth provider name
  clientId: string; // Client ID
  clientSecret: string; // Client secret
  scopes: string[]; // OAuth scopes
  redirectUri?: string; // Redirect URI (optional)
}

type PluginHook = (client: MCPClient) => Promise<void> | void;
```

**Note**: Tools listed in the `tools` array must be implemented on the Integrate MCP server. Plugins only configure OAuth and enable access to existing server-side tools.

## Lifecycle Hooks

Plugins support four lifecycle hooks:

### onInit

Called when the plugin is created (during `createMCPClient`):

```typescript
async onInit(client) {
  // Initialize plugin state
  console.log('Plugin initialized');
}
```

### onBeforeConnect

Called before establishing the connection to the MCP server:

```typescript
async onBeforeConnect(client) {
  // Prepare for connection
  console.log('About to connect');
}
```

### onAfterConnect

Called after successfully connecting to the MCP server:

```typescript
async onAfterConnect(client) {
  // Verify tools are available
  const tools = client.getEnabledTools();
  console.log('Tools available:', tools.length);
}
```

### onDisconnect

Called when disconnecting from the server:

```typescript
async onDisconnect(client) {
  // Cleanup
  console.log('Disconnected');
}
```

## Example: Slack Plugin

Here's a complete example of a Slack plugin:

```typescript
import { genericOAuthPlugin } from "integrate-sdk";

export const slackPlugin = (config: {
  clientId: string;
  clientSecret: string;
  scopes?: string[];
}) =>
  genericOAuthPlugin({
    id: "slack",
    provider: "slack",
    clientId: config.clientId,
    clientSecret: config.clientSecret,
    scopes: config.scopes || [
      "chat:write",
      "channels:read",
      "channels:manage",
      "users:read",
    ],
    tools: [
      "slack_send_message",
      "slack_list_channels",
      "slack_get_channel",
      "slack_create_channel",
      "slack_invite_user",
      "slack_list_users",
    ],
  });

// Usage
const client = createMCPClient({
  plugins: [
    slackPlugin({
      clientId: process.env.SLACK_CLIENT_ID!,
      clientSecret: process.env.SLACK_CLIENT_SECRET!,
    }),
  ],
});
```

## Discovering Available Tools

Before configuring a plugin, check what tools are available on the Integrate MCP server:

```typescript
// Create client and connect to discover available tools
const client = createMCPClient({
  plugins: [],
});

await client.connect();

const allTools = client.getAvailableTools();
console.log("All available tools:");
allTools.forEach((tool) => {
  console.log(`- ${tool.name}: ${tool.description}`);
});

// List tools by integration
const githubTools = await client.server.listToolsByIntegration({
  integration: "github",
});
console.log("GitHub tools:", githubTools);
```

## Calling Tools

For integrations configured with `genericOAuthPlugin` or `createSimplePlugin`, use the `_callToolByName` method:

```typescript
// Call a tool by its full name
const result = await client._callToolByName("slack_send_message", {
  channel: "#general",
  text: "Hello!",
});
```

For built-in plugins like GitHub and Gmail, use the typed methods:

```typescript
// Built-in plugins have typed methods
await client.github.createIssue({ owner: "user", repo: "repo", title: "Bug" });
await client.gmail.sendEmail({ to: "user@example.com", subject: "Hello" });
```

## Best Practices

1. **Verify tools exist** - Check that tools exist on the server before configuring them
2. **Use meaningful IDs** - Plugin IDs should match the server's integration IDs
3. **Provide sensible defaults** - Set default scopes for common use cases
4. **Document required scopes** - Let users know what OAuth permissions are needed
5. **Handle errors gracefully** - Use lifecycle hooks to validate configuration
6. **Test with the server** - Verify tools work as expected with the actual MCP server
